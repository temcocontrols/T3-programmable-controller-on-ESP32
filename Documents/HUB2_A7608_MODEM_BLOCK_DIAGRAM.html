Functional view of the HUB2 modem architecture: ESP32-S3 controls A7608E-H through UART AT commands, exposes status through Temco firmware, and keeps W5500 Ethernet available for T3000, Modbus TCP, and BACnet/IP. Current bring-up uses the LilyGo-Modem-Series-compatible pin map and ATDebug firmware.
Routing note: first prototype target is connect-out cellular data and modem status reporting. True 4G-to-Ethernet router behavior requires PPP, NAT, and IP forwarding and should be planned separately.

Design notes

  • Keep A7608 AT command handling inside one driver module.
  • Verify hardware with a small AT debug project before merging into the Temco firmware.
  • Do not make router behavior a first-prototype requirement.
  • W5500 and modem tasks must run together without starving Modbus TCP, BACnet/IP, or T3000 discovery.
  • Power behavior during LTE registration and data TX is a ship-critical test item.
